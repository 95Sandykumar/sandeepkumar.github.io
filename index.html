01 / The short story

Ambiguous requirements in, deployed agent systems out.

At Becker's Healthcare I build the AI layer end to end: an 11-persona multi-agent orchestration framework in TypeScript with a 4-tier intent-dispatch router, ~25 commands, an error/feedback learning loop, and a manual-approval human-in-the-loop gate. Every agent tool call passes a deterministic guardrail layer that validates it against hard-coded business rules before execution — it has blocked 156 invalid actions in production.

The rigor underneath it is the point. I built an LLM eval harness from 15 hand-labeled expert trajectories, scored multi-turn runs with DeepEval and LLM-as-judge, tracked everything in MLflow, and gated deployment on pass rate — catching 11% of regressions before release. A/B testing two tool-config variants shipped the winner on a 17-point lift; a LoRA fine-tune took eval accuracy from 55% to 83%. Governance (ISO 42001, NIST AI RMF) is wired in as automated guardrail enforcement, not paperwork.

I like the whole stack. RAG pipelines in n8n (Apify scraping, Pinecone embeddings, an LLM style-validation gate); a hermes-agent multi-LLM framework on AWS EC2 reading Snowflake and a knowledge graph; and the Snowflake + dbt medallion platform (25.5M+ events, 186K articles) that feeds it all. On the side: a live multi-model LLM audit engine with per-call failover, and a production RAG SaaS for purchase-order extraction with a self-training memory layer.
